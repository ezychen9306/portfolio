# 🔧 修复 Pages 设置 404 错误

## 问题

访问 `github.com/Enzhao-Chen/portfolio/settings/pages` 时显示 404。

## 可能的原因

1. **仓库名称不对**
   - 实际仓库名可能不是 `portfolio`
   - 检查你的仓库列表

2. **用户名不对**
   - GitHub 用户名可能不是 `Enzhao-Chen`
   - 检查你的实际用户名

3. **仓库不存在**
   - 仓库可能还没创建成功
   - 或已被删除

4. **权限问题**
   - 没有访问 Settings 的权限

## 解决步骤

### 步骤 1: 确认仓库信息

1. **访问你的 GitHub 主页**
   - https://github.com/Enzhao-Chen
   - 或直接登录 GitHub

2. **找到 portfolio 仓库**
   - 在你的仓库列表中查找
   - 确认仓库名称和用户名

3. **进入仓库**
   - 点击 `portfolio` 仓库
   - 确认能正常访问

### 步骤 2: 从仓库页面进入 Settings

**不要直接访问 settings/pages URL，而是：**

1. **进入仓库主页**
   - https://github.com/Enzhao-Chen/portfolio
   - 或从你的仓库列表进入

2. **点击 Settings 标签**
   - 在仓库页面顶部，有一排标签：Code, Issues, Pull requests, Actions, Projects, Wiki, Security, **Settings**
   - 点击 **Settings**

3. **找到 Pages**
   - 在 Settings 页面左侧菜单
   - 向下滚动找到 **Pages**
   - 点击 **Pages**

### 步骤 3: 如果仓库不存在

如果找不到 `portfolio` 仓库：

1. **检查是否创建成功**
   - 在 GitHub Desktop 中查看
   - 或访问你的仓库列表

2. **重新创建仓库**
   - 如果不存在，需要重新创建
   - 使用 GitHub Desktop 的 "Publish repository"

## 🎯 正确的访问方式

**不要直接输入 URL，而是：**

1. 登录 GitHub
2. 进入你的仓库列表
3. 找到 `portfolio` 仓库
4. 点击进入仓库
5. 点击顶部的 **Settings** 标签
6. 在左侧菜单找到 **Pages**
